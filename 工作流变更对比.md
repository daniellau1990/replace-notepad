# 工作流变更对比

> 变更日期: 2026-06-27
> 当前版本: v0.3.13

---

## 一、工作流对比

### 1.1 结构对比

| 维度 | 旧 | 新 |
|------|----|----|
| 工作流轨数 | **1 条**（所有任务走同样 9 步） | **2 轨**（Feature 8 步 / Bug Fix 7 步） |
| 代码门禁 | **机械阻断**：无计划文件 → Write/Edit 被 hook 拦截 | **对话驱动**：技能保证质量，hook 不管代码写入 |
| 最小流程 | 必须 brainstorm → propose → plan → code | Bug Fix ≤2 行可跳过诊断和计划，grill → 确认 → 修复 |

### 1.2 步骤对比

#### Feature 轨道

| 步骤 | 旧 | 新 | 变更 |
|------|----|----|------|
| 需求澄清 | `superpowers-skills-brainstorming` | `mattpocock-skills-grilling` | 替换为结构化追问 |
| 规格设计 | `/opsx:propose`（proposal + delta spec + design + tasks） | `mattpocock-skills-domain-modeling` | 轻量化，聚焦领域建模 |
| 计划 | `superpowers-skills-writing-plans` | `superpowers-skills-writing-plans` | 不变 |
| 人工审查 | 有 | 有 | 不变 |
| 实现 | `superpowers-skills-subagent-driven-development` | `mattpocock-skills-tdd` + 同上 | 增加 TDD 前置 |
| 代码审查 | **无** | `mattpocock-skills-review` | **新增** |
| 测试 | 双层（TDD + 回归） | 回归（TDD 已在实现阶段完成） | 简化为单层回归 |
| 归档 | `/opsx:archive` | `openspec-archive-change` + git commit + Version.md | 合并归档与版本 |
| 版本 | git commit + Version.md | 并入归档步骤 | 合并 |
| 调试 | `my-systematic-debugging` + `superpowers-skills-systematic-debugging` | 移至 Bug Fix 轨道 | 独立成 Bug Fix 流程 |

#### Bug Fix 轨道（新增）

| 步骤 | 技能 | 说明 |
|------|------|------|
| 1. 需求界定 | `mattpocock-skills-grilling` | 精确问题现象和边界 |
| 2. 诊断根因 | `mattpocock-skills-diagnosing-bugs` | 可跳过（≤2 行且根因明显） |
| 3. 制定计划 | `superpowers-skills-writing-plans` | 可跳过 |
| 4. 人工审查 | — | 用户确认 |
| 5. TDD 修复 | `mattpocock-skills-tdd` | 先复现测试 → 修复 → 验证 |
| 6. 回归测试 | `pytest tests/` | |
| 7. 归档 + 版本 | git commit + Version.md | |

## 二、Hook 对比

### 2.1 功能变化

| 维度 | 旧 | 新 |
|------|----|----|
| Gate 1: 计划文件门禁 | Write/Edit 目标为 `notepad/src/**/*.py` 时，检查 `docs/plans/YYYY-MM-DD-*.md` 是否存在 | **已删除** |
| Gate 2: Version.md 一致性 | git commit 时检查 Version.md 包含 APP_VERSION | **保留** |
| 白名单目录 | `docs/`、`.claude/`、根目录 bat/md 等 | **不需要了**（Gate 1 已删除） |

### 2.2 代码对比

| 指标 | 旧 | 新 |
|------|----|----|
| 代码行数 | 211 行 | 67 行 |
| 函数 | 7 个（`get_app_version`, `check_version_md`, `has_plan_for_today`, `classify_path`, `get_affected_paths`, `main` + 常量） | 3 个（`get_app_version`, `check_version_md`, `main`） |
| 拦截范围 | PreToolUse: Write、Edit、Bash(git commit) | PreToolUse: Bash(git commit) 仅 Version.md 检查 |
| 路径分类逻辑 | 有（区分 source/trivial/unknown） | **已删除** |
| 子进程调用 | `git diff --cached` 获取变更文件 | **已删除** |

### 2.3 阻断信息对比

**旧（Gate 1）**：
```
============================================================
 WORKFLOW CHECK: Action BLOCKED (no plan file)
============================================================
 Source files affected: src/app.py
 No plan file found in: docs/plans
 Today's date: 2026-06-27

 Required workflow steps before code changes:
   Step 1: Brainstorming -> Skill("superpowers-skills-brainstorming")
   Step 2: Proposal        -> Skill("openspec-propose")
   Step 3: Human review    -> ask user to confirm
   Step 4: Write plan      -> Skill("superpowers-skills-writing-plans")
            -> saves to docs/plans/2026-06-27-<feature>.md

 Create a plan file first, then retry.
============================================================
```

**新（仅 Gate 2）**：
```
============================================================
 WORKFLOW CHECK: Action BLOCKED (Version.md out of sync)
============================================================
 Current APP_VERSION: vX.Y.Z
 Version.md does NOT contain this version.

 Before committing, update notepad/Version.md:
   - Add a new ## vX.Y.Z section with change summary
   - Keep the format consistent with existing entries
   - The version string "vX.Y.Z" must appear in the file

 See CLAUDE.md "HOOK 保护" section.
============================================================
```

## 三、关键决策记录

| 决策 | 理由 |
|------|------|
| 删除计划文件门禁 | Hook 挡不住坏代码（可通过空文件绕过），只挡住快速修 bug 的人。行为规范由 grilling/review 技能对话保证 |
| 保留 Version.md 门禁 | 低摩擦、高价值、无法绕过，防止版本号遗忘 |
| Brainstorm → Grilling | Grilling 结构化追问更精准地挖出边界条件和隐含假设 |
| 新增 Code Review 步骤 | 弥补旧流程"实现直接到测试"的审查缺口 |
| 新增 Bug Fix 独立轨道 | 小修小补不需要全套 Feature 流程，降低流程负担 |
